# MoNa-pi 개발 진행 현황 (2026-04-15)

## 배경

기존 MoNaVLA (Kosmos-2 + 9-class discrete action)의 한계를 확인하고,
**π0 아키텍처 + Flow Matching** 기반의 연속 제어 시스템으로 재설계.

이번 세션에서 **데이터 수집 → 전처리 → 학습 → 추론 → 로봇 서버** 전체 파이프라인 골격을 구축함.

---

## 구현 완료 항목 (코드 존재, import 검증 완료)

### 1. 데이터 전처리 파이프라인 `data/preprocessing.py`

| 클래스 | 기능 | 출처 |
|--------|------|------|
| `ActionSmoother` | Savitzky-Golay 필터 (window=11, poly=3) — Bang-bang 스파이크 제거 | 신규 |
| `ActionNormalizer` | [-1.15, 1.15] → [-1.0, 1.0] 선형 스케일링 | 기존 dataset.py 이관 |
| `IntentPrefixInjector` | 9-class 의도 자동 분류 + 한/영 다변형 instruction 생성 (20% 노이즈) | MobileVLA-R1 방식 + MoNaVLA 계승 |
| `CounterfactualInjector` | stop_prob/steer_prob 확률로 액션+instruction 강제 오버라이드 | MoNaVLA `nav_h5_dataset_impl.py` 이관 |
| `DeltaActionConverter` | 속도 → 변위 변환 (옵션) | 신규 |
| `EpisodePreprocessor` | 위 모듈들을 묶는 파이프라인 | 신규 |

**검증**: `ActionSmoother`, `IntentPrefixInjector`, `CounterfactualInjector` 단위 테스트 통과

### 2. 데이터셋 `data/dataset.py`

기존 `ActionChunkDataset` 대폭 개선:

- **CLIP 정규화** 적용 (`mean=[0.485...], std=[0.269...]`) — 기존 단순 `/255.0`에서 교체 (MoNaVLA 계승)
- **v5/v6 포맷 자동 감지** (`observations/images` vs `images` 키)
- **HFlip 증강**: 이미지 반전 + `angular_z` 부호 반전 + instruction left↔right 교체 (MoNaVLA 계승)
- **ColorJitter** 증강 옵션 (`brightness/contrast/saturation/hue`)
- **CounterfactualInjector** 통합
- 경로 하드코딩 제거 → 생성자 파라미터화
- `build_train_val_split()`: train/val 분리 시 증강은 train만 적용

### 3. 학습 설정 `configs/train.yaml`

```
model: action_dim=3, horizon=10, hidden_dim=512
data:  v5 경로, val_split=0.1, augment/counterfactual 플래그 포함
train: batch=4, lr=1e-4, epochs=50, warmup=500, grad_clip=1.0
```

### 4. 학습 루프 `training/train.py`

기존 스켈레톤 → 완전한 루프로 재작성:

- `train_one_epoch()` / `validate()` 분리
- **Cosine LR + Warmup** 스케줄러
- **W&B / TensorBoard** 선택 로깅
- **Top-k 체크포인트** 관리 (best val_loss 자동 저장)
- Hydra 없이 YAML 직접 로딩 (의존성 최소화)

### 5. 오프라인 평가 `training/evaluate.py`

- Normalized MSE / Raw MSE / Per-dim MSE (lx, ly, az) 출력
- `python training/evaluate.py --config configs/train.yaml --ckpt checkpoints/best`

### 6. 추론 모듈 `inference/`

| 파일 | 내용 |
|------|------|
| `ode_solver.py` | EulerSolver / HeunSolver / DPMSolver (import 검증 완료) |
| `engine.py` | `MoNaPiEngine` — 비전 특징 캐싱(TTL 0.5s), FP16 일관성 |
| `server.py` | FastAPI REST (`POST /predict`, `GET /health`, `GET /metrics`) |

**현재 상태**: 코드 작성 및 import 완료. 실제 모델 체크포인트 없어 end-to-end 실행 미검증.

### 7. 로봇 서버 `robot/`

| 파일 | 내용 |
|------|------|
| `action_buffer.py` | `ActionChunkBuffer` — Receding Horizon (50% 소진 시 replan 트리거) |
| `ros2_controller.py` | ROS2 노드 + `VLAControlManager` (MoNaVLA `vla_control_utils.py` 이관) |
| `launch/mona_pi.launch.py` | 추론 서버 + ROS2 컨트롤러 동시 기동 |

`VLAControlManager` 이관 내용:
- `publish_and_move()`: ROS Twist 발행 + pop 하드웨어 직접 제어
- `robust_stop()`: 5회 연속 정지 패킷 (버퍼 클리어)
- `move_and_stop_timed()`: 0.4s 이동 후 자동 정지

**현재 상태**: ROS2 환경 없어 실행 미검증. 구조 완성.

### 8. 데이터 수집 `collect/`

| 파일 | 내용 |
|------|------|
| `keyboard_collector.py` | `RampController` (이차 가속도 — 0.5s에 max 1.15 도달) + HDF5 v6 저장 |
| `ros2_recorder.py` | ROS2 `/cam_high/image_raw` + `/cmd_vel` 구독 → HDF5 저장 |

### 9. Shell 확장 `.mona_pi_aliases`

기존 `vla-*` alias를 `--pi` 파라미터로 MoNa-pi에서도 사용 가능:

```bash
vla-collect --pi --output /data/ep001.h5 --task "장애물 회피"
vla-train   --pi configs/train.yaml
vla-server  --pi --ckpt checkpoints/best
vla-inference --pi --instruction "장애물 피해 직진"
vla-eval    --pi checkpoints/best
vla-env     --pi
vla-help    --pi
```

`~/.zshrc` 맨 끝에 `source /home/soda/MoNa-pi/.mona_pi_aliases` 추가됨.
