# Domain Analysis: Manipulation (π0) vs Mobile Navigation (MoNaVLA)

## 1. Background
현재 π0(Pi-zero) 아키텍처를 모바일 로봇 내비게이션 태스크로 전이(Transfer)하는 과정에서 모델의 첫 추론 결과를 확인했습니다. 모델은 현재 사전학습된 VLM 백본만 로드되었고 Action Head 측은 무작위 초기화 상태이므로, 출력은 온전한 가우시안 노이즈(Gaussian Noise)에 가깝습니다. 본 문서에서는 이 무작위 출력의 원인과 함께, 향후 학습 성패를 가를 **매니퓰레이션 도메인과 모바일 내비게이션 도메인 간의 근본적 데이터 속성 차이(Domain Gap)**를 분석합니다.

## 2. Analysis
### 2.1 미학습 상태의 출력 해석 (Why is it random?)
*   **현상**: 모델 출력(Generated)은 [-2.8 ~ +2.7] 사이로 무작위 분산되어 있으며, 직진(`vx=1.150`)이나 정지(`0.000`) 같은 GT의 의도를 전혀 반영하지 못합니다.
*   **원인**: Flow Matching은 노이즈 $x_1 \sim \mathcal{N}(0, I)$에서 시작하여 조건부 벡터장(Vector field)을 따라 정답 $x_0$로 궤적을 굽히는(Deformation) 과정입니다. 현재 ODE Solver가 작동하고 코드는 정상이나, 이 궤적을 굽히는 'Flow Head'의 가중치가 학습(ODE Integration 학습 단계)되지 않아 단순히 임의의 분포를 띄는 노이즈를 반환하고 있을 뿐입니다.

### 2.2 도메인 별 Action Space 특성 비교
원본 논문의 로봇 팔(Manipulation)과 현재 MoNaVLA 모바일 구동 제어(Navigation) 간의 핵심적인 차이를 비교합니다.

| 속성 (Attribute) | 원본 π0 (Manipulation) | MoNa-pi (Mobile Navigation) | 도메인 변환 시 발생 가능한 이슈 |
| :--- | :--- | :--- | :--- |
| **제어 대상** | 로봇 팔 End-effector 위치/자세 (6-DoF), 그리퍼 | 3-옴니휠 모바일 로봇 속도 (3-DoF) | 위치 기반 모델 구조를 모터 전압/속도로 해석 |
| **물리적 단위** | 절대/상대 위치 (Position), 쿼터니언 (Rotation) | 선속도(vx, vy), 각속도(wz) | 속도 오차 누적으로 인한 위치 인식 편향 발생 위험 |
| **기준 좌표계** | 로봇 베이스(Base) 기준 고정 좌표 | 몸체 중심(Robot-centric) 상대 속도계 | 카메라 시점(뷰포인트)이 액션에 따라 계속해서 이동됨 |
| **데이터 분포 특성** | 부드러운 연속 확률 분포 (Smooth Continuous) | **이산적 및 Bang-bang 제어 특성** (예: 0, 1.15 등 지정값 쏠림) | 연속 생성 모델이 극단적인 1.15 상수값을 모사할 때 진동(Oscillation) 발생 |
| **시간/맥락 의존성** | 주로 단기적 (눈앞의 사물 당장 조작) | 시계열 흐름에 따른 장거리 회피 기동 (장기 의존성) | 직전 시점의 관성이나 시각적 기억력이 더 필요해질 수 있음 |

## 3. Findings

1.  **방향성을 잃은 것이 아니라 아직 '태어나지(학습되지)' 않은 것**:
    현재 모델은 이미지를 보아도 "그래서 이걸 보고 어떻게 움직이라는 거지?"라는 판단 기준(가중치)이 전무합니다. 그래서 난수(Noise) 상태를 그대로 내보내고 있으며, 이는 지극히 정상적인 초기 상태입니다.

2.  **데이터 분포 불일치 위험 (The Bang-Bang Control Problem)**:
    v5 데이터를 보면 로봇의 속도가 주로 `1.150` 구동 아니면 `0.000` 정지로 **온/오프(On/Off) 하우스 스위치처럼** 수집되었습니다. 매니퓰레이터의 부드러운 궤적을 학습하도록 설계된 Flow Matching 모델에게 이렇게 뚝뚝 끊기는 Bang-bang 데이터를 그대로 넣게 되면, 모델 입장에서는 확률 밀도가 중간이 비어버린 형태여서 예측에 큰 혼란을 겪게 됩니다.

## 4. Conclusion
현재 출력값은 지극히 정상적인 "백지" 상태입니다. 하지만 성공적인 도메인 전이를 달성하기 위해 당장 조치해야 할 사항은 명확합니다.

*   **Action Normalization (필수)**: `1.150` 등의 물리계 수치를そのまま 넣지 말고, 평균 0 / 표준편차 1 (혹은 Min-Max [-1, 1]) 범위로 **정규화**하여 스케일을 맞춰주어야 합니다. 1.15는 신경망 입장에서 너무 큰 값입니다.
*   **Smoothing (권장)**: 액션 시퀀스의 각이 진(Sharp) 속도 변화 구간을 모달리티 필터링 하여 조금 더 곡선에 가깝게 부드럽게(Smoothing) 가공해 주는 과정이 추가된다면 VLA 학습 안정성이 훨씬 높아질 것입니다.
